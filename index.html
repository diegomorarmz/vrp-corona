Documentación técnica — Motor Nearest Neighbor VRP
Esta sección describe cómo funciona el algoritmo implementado, sus decisiones de diseño y sus límites. Útil para presentaciones académicas y defensa técnica del proyecto.
1
Ordenamiento por prioridad y distancia
Antes de construir rutas, los clientes se ordenan: primero por prioridad (alta → media → baja) y, dentro de cada nivel, por distancia ascendente al depósito. Esto garantiza que los clientes críticos —por ejemplo, tiendas de conveniencia con ventana estrecha— sean atendidos al inicio de la jornada, cuando el camión aún tiene plena capacidad.
2
Construcción greedy — Nearest Neighbor
El algoritmo itera sobre la lista ordenada y asigna cada cliente a la ruta activa mientras no se viole ninguna restricción. Para cada nuevo cliente, evalúa dos condiciones de quiebre:

Restricción de capacidad: si agregar la demanda del cliente supera la capacidad del vehículo (cajas), se cierra la ruta y se abre una nueva.
Restricción de jornada: si el tiempo acumulado más el tiempo estimado del cliente supera la jornada máxima del operador (en horas), también se cierra y abre ruta nueva.
para cada cliente en clientes_ordenados:
  // Estima tiempo: descarga + traslado fijo (8 min)
  tiempoCliente = cliente.descarga + 8
  
  si carga + cliente.demanda > capacidadMax:
    cerrarRuta() → abrirRutaNueva()
  si minutosAcumulados + tiempoCliente > jornadaMax × 60:
    cerrarRuta() → abrirRutaNueva()
  
  agregarCliente(rutaActual)
3
Cálculo de costo por ruta
Al cerrar cada ruta, el motor calcula el costo real combinando tres componentes independientes: kilometraje, combustible y tiempo del operador. Esta separación permite comparar dónde está el mayor costo y orientar la optimización.
costo = (km × costo_por_km)
      + (horas × costo_operador_hora)
      + (km ÷ rendimiento_km_litro) × precio_gasolina
4
Comparativa con operación actual
El resultado final se contrasta contra el promedio real de rutas ingresadas en el historial. Si hay datos registrados, se usa el promedio real de km y costo; si no, se usan valores de referencia del sector (125 km, $1,400/día). El ahorro mensual asume 22 días laborables.
Complejidad: O(n²)
Capacidad: hasta ~200 clientes
Tipo: Heurística constructiva greedy
Solución: Subóptima pero rápida
⚠ Limitaciones conocidas del algoritmo NN:
El vecino más cercano es una heurística, no un algoritmo exacto. En redes de 20+ clientes puede generar rutas hasta un 20–25% más largas que el óptimo global. Para producción real se recomendaría combinar NN con mejora local (2-opt), o sustituirlo por OR-Tools (Google) o Concorde para instancias grandes. Para el alcance de prácticas profesionales, NN ofrece un resultado bueno con implementación comprensible y demostrable.
